Introduction

Ollin est un langage de script minimaliste et expressif : dynamiquement typé, compilé, et conçu pour être embarqué.

Premier programme

print("hello world!")

Types

Règles de vérité (vrai/faux)

Principe unique : le vide est faux. Une valeur est fausse si elle représente l'absence ou le vide (nil, false, zéro, chaîne/map/array vide) ; tout le reste est vrai.

niltoujours

stringchaîne vide ""

assert(1 == true)   ## true est stocké comme 1
assert(0 == false)  ## false comme 0
assert("non vide")  ## chaîne non vide => vrai
assert(not "")      ## chaîne vide => faux

Arithmétique des types numériques

Deux entiers combinés donnent un entier — sauf la division /, qui donne toujours un flottant. Dès qu'un flottant intervient dans le calcul, le résultat est un flottant.
assert(7 / 2 == 3.5)  ## division => toujours float
assert(7 % 2 == 1)    ## modulo => int si les deux sont int

Variables

Toute variable doit être déclarée avant utilisation — utiliser ou affecter un nom non déclaré est une erreur de compilation.

var crée une variable locale à la fonction courante ; global crée une variable globale visible dans tout le programme.

Nommage

Un nom commence par une lettre ou un underscore, suivi de lettres, chiffres ou underscores (_compteur, x1, score_final). Sensible à la casse (score ≠ Score) ; les accents ne sont pas supportés. Les mots-clés du langage (var, if, end…) ne peuvent pas servir de nom.

var a = 10
var b = 2.5
var s = "bonjour"
var undef  ## pas d'initialisation => nil
assert(undef == nil)

Déclaration multiple

var x, y = 1, 2
var p, q, r = 10, 20  ## r => nil (pas assez de valeurs)

Affectation

var a = 10
a = 42  ## simple
a += 1  ## compound : += -= *= /= %=
a *= 2
assert(a == 86)
Les underscores dans les littéraux numériques sont ignorés : 1_000_000 == 1000000.

Variables globales — global

Le mot-clé global déclare une variable globale, visible dans tout le programme. Elle peut être déclarée n'importe où et lue ou modifiée depuis n'importe quelle fonction. Une variable locale (var) de même nom masque le global dans sa portée.

global compteur = 0

func incr()
    compteur += 1  ## modifie le global depuis une fonction
end

incr()
incr()
assert(compteur == 2)
print(compteur)    ## 2
var crée toujours une variable locale ; global crée une variable globale visible partout. Affecter un nom non déclaré reste une erreur de compilation.

Constantes — const

Le mot-clé const déclare une locale immuable. L'initialisation est obligatoire. Toute tentative d'assignation ultérieure est une erreur de compilation.

const PI = 3.14159
const MAX = 100

assert(PI == 3.14159)
assert(MAX == 100)

func aire(r)
    const DEUX_PI = 2 * PI
    return DEUX_PI * r * r
end

print(aire(1))  ## 6.28318…
Les erreurs suivantes sont détectées à la compilation : 
const x  ## ERROR: must be initialized
const x = 1
x = 2    ## ERROR: cannot assign to const 'x'

Opérateurs bits

Les opérateurs bit-à-bit opèrent uniquement sur des entiers. Le ~ est XOR en binaire et NOT en unaire ; ^ est réservé à la puissance.

Binaires

OpDescriptionExemple
&ET bit-à-bit12 & 10 == 8
|OU bit-à-bit12 | 10 == 14
~XOR bit-à-bit12 ~ 10 == 6
<<Décalage gauche1 << 3 == 8
>>Décalage droit (arithmétique)16 >> 2 == 4

Unaire

OpDescriptionExemple
~NOT (complément à 1)~0 == -1, ~1 == -2
Le ~ est XOR entre deux opérandes (a ~ b) et NOT en préfixe (~a). ^ n'est pas un opérateur bit-à-bit : c'est la puissance (voir Opérateurs).

Précédence

De la plus basse à la plus haute :

or  =>  and  =>  |  =>  ~(xor)  =>  &  =>  comparaisons  =>  << >>  =>  + -  =>  * / %  =>  ~(not) - not  =>  ^

Exemples

var flags = 0
flags = flags | 1         ## activer bit 0
flags = flags | 4         ## activer bit 2
assert(flags == 5)        ## 0101

assert((flags & 1) == 1)  ## tester bit 0
assert((flags & 2) == 0)  ## bit 1 absent

flags = flags ~ 1         ## XOR : basculer bit 0 => 4
assert(flags == 4)

assert(1 << 8 == 256)
assert(256 >> 4 == 16)
assert(~0 == -1)
assert(255 & 15 == 15)
Les opérateurs bit-à-bit nécessitent des opérandes entiers (integer). Appliquer &, |, ~, << ou >> à un float lèvera une erreur à l'exécution.

Chaînes

var s = "bonjour"
assert(s == "bonjour")
assert(not "")   ## chaîne vide est fausse
assert("texte")  ## non vide est vrai

La concaténation se fait avec + :

var greeting = "Hello, " + "world" + "!"
assert(greeting == "Hello, world!")
Les chaînes sont immuables. Seul + est surchargé pour les strings — les autres opérateurs (-, *…) restent réservés aux nombres.

Expressions multi-lignes

Les sauts de ligne sont traités comme de l'espace blanc — une expression peut s'étendre sur autant de lignes que nécessaire, sans syntaxe spéciale.

var total =
    prix_ht
    * ((1 + tva) / 100)
    + frais_port

if commande_valide
    and stock_disponible
    and paiement_ok then
    expédier()
end
then, do et end délimitent les blocs — les parenthèses autour des conditions ne sont jamais obligatoires.

If / Else

if condition then
    ## bloc then
elseif autre_condition then
    ## elseif (autant que nécessaire)
else
    ## else optionnel
end
var score = 75
var grade = "F"

if score >= 90 then
    grade = "A"
elseif score >= 80 then
    grade = "B"
elseif score >= 70 then
    grade = "C"
else
    grade = "F"
end

assert(grade == "C")
print(grade)  ## C

While

var i = 0
while i < 10 do
    i += 1
end
assert(i == 10)

## break quitte la boucle immédiatement
var k = 0
while true do
    k += 1
    if k >= 5 then break end
end
assert(k == 5)

For

Deux variantes pour les boucles numériques : par intervalle (for i in [a;b]) ou par bornes explicites (for i=a,b). Pour l'itération sur les maps et les arrays, voir Itération sur une map et Arrays.

Range [a;b]

Notation d'intervalles mathématiques — bornes inclusives ou exclusives, step optionnel.

var sum = 0
for i in [1;10] do
    sum += i
end
assert(sum == 55)  ## 1+2+…+10 (inclusif des deux côtés)

## droit exclusif : 1,2,...,9
for i in [1;10[ do
    print(i)
end

## avec step : 1,3,5,7,9
for i in [1;10;2] do
    print(i)
end

## first-class : stockable dans une variable
var r = [1;5]
for i in r do
    print(i)       ## 1 2 3 4 5
end

Numérique i=start,end[,step]

Step optionnel — positif ou négatif.

## step implicite = 1
for i=1,5 do
    print(i)  ## 1 2 3 4 5
end

## step positif (pairs)
for i=0,10,2 do
    print(i)  ## 0 2 4 6 8 10
end

## step négatif (compte à rebours)
for i=5,1,-1 do
    print(i)  ## 5 4 3 2 1
end

Break dans for

for i in [1;100] do
    if i > 3 then break end
    print(i)  ## 1 2 3
end

Switch

switch compare une expression à plusieurs valeurs. Chaque branche commence par case suivi d'une ou plusieurs valeurs séparées par des virgules. Le bloc else s'exécute si aucun case ne correspond. break est valide à l'intérieur.

var x = 2
switch x
case 1
    print("un")
case 2, 3
    print("deux ou trois")
else
    print("autre")
end

Switch sur des chaînes

var cmd = "start"
switch cmd
case "start", "go"
    print("démarrage")
case "stop"
    print("arrêt")
else
    print("commande inconnue")
end

Fonctions

func add(a, b)
    return a + b
end

assert(add(3, 4) == 7)

Fonction sur un champ de map

func obj.champ(...) définit une fonction directement sur un champ de map ou de module — sucre pour obj.champ = func(...) ... end. Utile pour les hooks moteur (ex. keyboard.keypressed).

var obj = {}
func obj.salut(nom)
    return "salut " + nom
end
assert(obj.salut("ollin") == "salut ollin")

Retours multiples

func minmax(a, b)
    if a < b then
        return a, b
    end
    return b, a
end

var lo, hi = minmax(7, 3)
assert(lo == 3 and hi == 7)

Récursion

func fact(n)
    if n < 2 then return 1 end
    return n * fact(n - 1)
end

assert(fact(10) == 3628800)

Appel optionnel — f?()

La syntaxe f?(args) est exactement le sucre de if f then f(args) else nil :

  • f fausse (nil, 0, "", [], {}) → aucun appel, résultat nil, et les arguments ne sont pas évalués (aucun effet de bord) ;
  • f appelable → appel normal ;
  • f vraie mais non appelable (entier, chaîne…) → erreur d'exécution, comme un appel ordinaire.

Cas d'usage typique : callback facultatif (nil → ignoré sans rien exécuter). S'applique à toutes les formes d'appel, y compris les méthodes (obj.m?() injecte self ; méthode absente → nil) et super.m?(). Côté implémentation, c'est un simple JUMP_IF_FALSE suivi de l'appel standard — pas d'opcode dédié.

func greet() return "hello" end
assert(greet?() == "hello")  ## appelable => appel normal

var cb = nil
assert(cb?() == nil)         ## nil => pas d'appel, pas d'erreur

## callback facultatif + appel optionnel de méthode (self injecté)
class Box
    func init(v) self.v = v end
    func get() return self.v end
end
var b = Box(7)
assert(b.get?() == 7)        ## self injecté
assert(b.missing?() == nil)  ## méthode absente => nil
Une valeur vraie mais non appelable reste une erreur : (42)?() lève « call on non-function value » (42 est vraie → on tente l'appel). Seules les valeurs fausses sont ignorées. Le ? n'agit que sur l'appel immédiatement à sa gauche.

Arguments variadiques

... capture tous les arguments supplémentaires.

func log(niveau, ...)
    printf("[{}] ", niveau)
    print(...)
end

log("INFO", "démarrage", "ok")  ## [INFO] démarrage ok

Passer les varargs à une autre fonction

func wrapper(a, ...)
    return a, ...  ## propage les varargs
end

var x, y, z = wrapper(1, 2, 3)
assert(x == 1 and y == 2 and z == 3)

Paramètres par défaut

Les valeurs par défaut doivent être des constantes littérales.

func greet(name, greeting="Bonjour")
    printf("{}, {} !", greeting, name)
end

greet("Alice")         ## Bonjour, Alice !
greet("Bob", "Salut")  ## Salut, Bob !
func connect(host="localhost", port=8080, debug=false)
    printf("{}:{} debug={}", host, port, debug)
end

connect()                    ## localhost:8080 debug=0
connect("prod.example.com")  ## prod.example.com:8080 debug=0
connect("prod", 443)         ## prod:443 debug=0

Closures

Une fonction peut capturer les variables de la portée qui l'entoure. La valeur capturée reste vivante même après le retour de la fonction externe.

Compteur encapsulé

func make_counter()
    var n = 0
    func inc()
        n += 1
        return n
    end
    return inc
end

var c1 = make_counter()
var c2 = make_counter()  ## état indépendant

assert(c1() == 1)
assert(c1() == 2)
assert(c2() == 1)        ## c2 repart de 0

Fabrique de fonctions

func make_adder(x)
    func add(y)
        return x + y
    end
    return add
end

var add5  = make_adder(5)
var add10 = make_adder(10)

assert(add5(3)  == 8)
assert(add10(3) == 13)

Fonctions imbriquées

Une fonction déclarée à l'intérieur d'une autre reste locale — elle n'est pas ajoutée à l'environnement global.

func process(data)
    func helper(x)        ## visible uniquement dans process
        return x * 2
    end
    return helper(data) + helper(data + 1)
end

assert(process(3) == 14)  ## 3*2 + 4*2

État partagé entre fonctions

func make_shared()
    var count = 0
    func inc()  count += 1  end
    func get()  return count  end
    return inc, get
end

var inc, get = make_shared()
inc()
inc()
inc()
assert(get() == 3)
print(get())  ## 3

Maps

Dictionnaire clé→valeur. La syntaxe de littéral accepte des chaînes, des identifiants, ou des clés calculées [expr] ; à l'exécution, toute valeur peut servir de clé (nil, entier, float, string, bool, array, map, fonction…).

Création

var vide = {}

var point = {"x": 10, "y": 20}

## clés sans guillemets
var config = {
    host:  "localhost",
    port:  8080,
    debug: false,
}

Clés littérales et clés calculées

Une clé écrite en identifiant, en chaîne, ou entre crochets autour d'une chaîne est la même clé littérale :

var a = {ref: 12}      ## clé = la string "ref"
var b = {"ref": 12}    ## identique
var c = {["ref"]: 12}  ## identique

Entre crochets, une expression est évaluée à l'exécution et sa valeur devient la clé :

var ref = "couleur"
var m = {[ref]: "rouge"}     ## clé = valeur de ref => "couleur"
assert(m["couleur"] == "rouge")
assert(m["ref"]     == nil)  ## "ref" littéral n'est PAS une clé

## la clé calculée peut être de tout type
var n = {[1 + 1]: "deux"}
assert(n[2] == "deux")

La même règle vaut pour l'accès : entre crochets, un littéral string désigne la clé telle quelle, une variable désigne sa valeur. Lecture, écriture et affectation composée sont identiques :

var ref = "couleur"
var m = {}
m[ref]     = "rouge"              ## écrit le slot "couleur" (valeur de ref)
m["taille"] = 10                  ## écrit le slot "taille" (littéral)
assert(m["couleur"] == "rouge")   ## relu via littéral
assert(m.couleur     == "rouge")  ## ou via point
m["taille"] += 5
assert(m["taille"] == 15)

Lecture

Crochets
point["x"]  ## 10
point["z"]  ## nil
Point (équivalent)
point.x  ## 10
point.z  ## nil

Écriture

var point = {"x": 10, "y": 20}
point["z"] = 0   ## nouvelle clé
point.z = 5      ## équivalent

point["x"] += 1  ## compound via crochet
point.y *= 2     ## compound via point

assert(point.x == 11)
assert(point.y == 40)

Maps imbriquées

var scene = {
    camera: {fov: 60, near: 0.1, far: 1000},
    light:  {x: 0, y: 10, z: 0, intensity: 1},
}

assert(scene["camera"]["fov"] == 60)

Clés de tout type (via crochets)

var m = {}

m[nil]   = "nil key"
m[42]    = "int key"
m[3.14]  = "float key"
m[true]  = "bool key"

## 1 et 1.0 sont la même clé
m[1] = "one"
assert(m[1.0] == "one")

Sémantique de copie

Les maps sont partagées par référence (comptage de référence). Affecter une map à une autre variable ne la clone pas.
var a = {x: 1}
var b = a          ## b pointe vers la même map
b.x = 99
assert(a.x == 99)  ## a est modifié aussi
Conséquence : == compare les maps par référence, pas par contenu — {} == {} vaut 0. Pour une égalité par valeur, voir Égalité et comparaison.

Itération sur une map

La syntaxe for k, v in m parcourt toutes les entrées de la map. L'ordre n'est pas garanti.

var scores = {Alice: 95, Bob: 82, Carol: 78}

for nom, score in scores do
    printf("{} => {}", nom, score)
end
## Alice => 95
## Bob => 82
## Carol => 78

Somme des valeurs

var scores = {Alice: 95, Bob: 82, Carol: 78}
var total = 0
for k, v in scores do
    total += v
end
printf("moyenne : {}", total / 3)

Dans une fonction

func sum_values(m)
    var s = 0
    for k, v in m do
        s += v
    end
    return s
end

var scores = {Alice: 95, Bob: 82, Carol: 78}
assert(sum_values(scores) == 255)

Arrays

Tableau dynamique distinct des maps. Les indices commencent à 1. Comme les maps, les arrays sont partagés par référence.

Création

var vide = []

var t = [10, 20, 30]

var calc = [1+1, 2*3, 10-1]  ## expressions autorisées

Lecture / Écriture

print(t[1])  ## 10  (indexé à 1)
print(t[4])  ## nil (hors bounds => nil)

t[2] = 99    ## set
t[2] += 1    ## compound : += -= *= /= %=

var a = []
a[5] = "x"   ## grossit automatiquement (indices 1..4 => nil)

Itération — valeurs seules

var sum = 0
for v in [1, 2, 3, 4, 5] do
    sum += v
end
assert(sum == 15)

Itération — index + valeur

var t = ["a", "b", "c"]
for i, v in t do
    printf("{}: {}", i, v)
end
## 1: a
## 2: b
## 3: c

Sémantique de copie

Les arrays sont partagés par référence (comptage de référence), comme les maps.

Gestion d'erreurs

Ollin utilise try / catch / else / end. Le bloc else est exécuté si aucune exception n'est levée. Tous les blocs sont optionnels.

try
    throw "quelque chose s'est mal passé"
catch err
    print("erreur :", err)
else
    print("tout s'est bien passé")
end

Throw de n'importe quel type

func diviser(a, b)
    if b == 0 then
        throw {code: 42, msg: "division par zéro"}
    end
    return a / b
end

try
    var r = diviser(10, 0)
catch e
    printf("erreur {}: {}", e["code"], e["msg"])
end

Try sans corps

try
catch err
    print(err)
end

Fonctions intégrées

print

Affiche tous les arguments séparés par des espaces, suivi d'un retour à la ligne.

print("a =", 42, true)  ## a = 42 1
print()                 ## ligne vide

printf

Formatage avec {} pour substitution positionnelle ou {n} pour indexation explicite.

printf("{} + {} = {}", 1, 2, 3)      ## 1 + 2 = 3
printf("{0} et {0}", "oui")          ## oui et oui
printf("x={0} y={1} x={0}", 10, 20)  ## index explicite

assert

Lève une exception si la condition est fausse. Message optionnel.

var x = 42
assert(1 + 1 == 2)
assert(x > 0, "x doit être positif")

time

Retourne le temps UNIX en secondes (float). Utile pour les benchmarks.

var t0 = time()

## ... code à mesurer ...

var t1 = time()
printf("durée : {} s", t1 - t0)

Exemple complet

## Benchmark : somme de 1 million d'itérations
var N = 1_000_000
var t0 = time()

var s = 0
for i in [1;N] do
    s += i
end

var t1 = time()
printf("somme = {}  temps = {} s", s, t1 - t0)

len

Retourne la longueur d'un array, d'une map, d'une string ou d'un range (nombre d'éléments).

len([1, 2, 3])  ## 3
len("bonjour")  ## 7
var m = {"a": 1, "b": 2}
len(m)          ## 2
len([1;10])     ## 10 (range inclusif)

typeof

Retourne le type d'une valeur sous forme de string : "nil", "int", "float", "string", "array", "map", "function", "class", "range".

typeof(42)       ## "int"
typeof(3.14)     ## "float"
typeof("hello")  ## "string"
typeof([1, 2])   ## "array"
typeof(nil)      ## "nil"

Fonctions intégrées comme valeurs

print, printf, assert, time, len et typeof sont des valeurs de première classe — assignables, passables en argument :

var f = print
f("hello")        ## hello

func apply(fn, x)
    fn(x)
end
apply(print, 42)  ## 42

Import

L'instruction import charge un fichier Ollin et rend son contenu disponible dans le programme courant.

Les chemins sont relatifs au répertoire du script courant. L'extension .ol est ajoutée automatiquement si absente.

Import plat

Toutes les définitions du fichier importé deviennent disponibles directement dans le scope courant.

## utils.ol
func add(a, b)
    return a + b
end
var PI = 3.14159

## main.ol
import "utils"
print(add(2, 3))  ## 5
print(PI)         ## 3.14159

Import modulaire

Avec as name, toutes les définitions sont regroupées dans une map nommée.

import "utils" as math
print(math.add(10, 20))  ## 30
print(math.PI)           ## 3.14159

Règles

  • Chemins résolus relativement au répertoire du script courant (récursivement pour les imports imbriqués).
  • Les imports circulaires sont détectés et ignorés (cache partagé).
  • Un même fichier importé deux fois n'est exécuté qu'une fois.

Classes

Ollin supporte l'orienté-objet avec héritage simple, constructeur implicite et méta-méthodes.

Déclaration

class Animal
    func init(name, sound)
        self.name = name
        self.sound = sound
    end
    func speak()
        return self.name + " says " + self.sound
    end
    func __str()
        return "Animal(" + self.name + ")"
    end
end

var a = Animal("Rex", "Woof")
print(a)            ## Animal(Rex)   — affichage de l'instance via __str
print(a.speak())    ## Rex says Woof
  • func init(...) est le constructeur ; self est passé implicitement en premier paramètre.
  • Toutes les méthodes reçoivent self automatiquement.
  • func __str() définit l'affichage de l'instance : print(obj) et la concaténation "…" + obj l'utilisent.

Instanciation

class Animal
    func init(name, sound)
        self.name = name
        self.sound = sound
    end
    func speak()
        return self.name + " says " + self.sound
    end
end
var a = Animal("Dog", "woof")
print(a.speak())   ## Dog says woof
print(a.name)      ## Dog

Méthodes statiques

Une méthode préfixée par static appartient à la classe, pas à une instance : elle ne reçoit pas self. On l'appelle via Classe.methode(...) (ou sur une instance, sans injection de self). Utile pour les constructeurs alternatifs ou les utilitaires liés à la classe.

class Vec
    func init(x, y)
        self.x = x
        self.y = y
    end
    static func zero()      ## pas de self : fabrique liée à la classe
        return Vec(0, 0)
    end
end
var o = Vec.zero()          ## appel sur la classe
assert(o.x == 0 and o.y == 0)

Héritage

class Animal
    func init(name, sound)
        self.name = name
        self.sound = sound
    end
    func speak()
        return self.name + " says " + self.sound
    end
end
class Dog extends Animal
    func init(name)
        super.init(name, "woof")   ## appel du constructeur parent
    end
    func fetch()
        return self.name + " fetches!"
    end
end

var d = Dog("Rex")
print(d.speak())   ## Rex says woof  (méthode héritée)
print(d.fetch())   ## Rex fetches!

Si un enfant n'a pas de func init, il hérite automatiquement de celui du parent.

super.method(args) appelle la méthode du parent avec le self courant.

Égalité et comparaison

Les maps et arrays nus se comparent par référence : a == b n'est vrai que si a et b désignent le même objet en mémoire, jamais d'après leur contenu. Pour une comparaison par valeur, il faut en faire une classe et définir __eq.

## maps nues => comparaison par référence
print({x: 1} == {x: 1})            ## 0 : contenu identique mais objets distincts
var m = {x: 1}
print(m == m)                      ## 1 : même référence

## instance + __eq => comparaison par valeur
class Point
    func init(x, y)
        self.x = x
        self.y = y
    end
    func __eq(other)
        return self.x == other.x and self.y == other.y
    end
end
print(Point(1, 2) == Point(1, 2))  ## 1 : __eq compare les champs
print(Point(1, 2) == Point(3, 4))  ## 0
Deux subtilités sur __eq : il n'est cherché que sur l'opérande de gauche (si seule la droite est une instance, la comparaison reste par référence) ; et il est appelé même quand les deux opérandes sont la même instance — il n'y a pas de court-circuit d'identité.

Graphics

Le module graphics expose Raylib pour le rendu 2D en temps réel. Il est disponible automatiquement sans import. Le rendu fonctionne sur desktop et WASM.

Cycle de vie

Trois fonctions optionnelles, appelées automatiquement par le moteur : setup() une fois au démarrage, puis update(dt) et draw() à chaque frame.

## appelée UNE FOIS après le chargement, avant la boucle
func setup()
    graphics.canvas(800, 600, "Titre")
end

## appelée automatiquement à chaque frame
func draw()
    graphics.clear(colors.BLACK)

    ## dessin ici

end

Ces trois fonctions sont indépendantes : tu peux n'en définir qu'une. (Tu peux aussi ouvrir la fenêtre au niveau du script plutôt que dans setup.)

Pour séparer la logique du rendu, définis aussi update(dt) : le moteur l'appelle avant draw à chaque frame, avec dt = nombre de secondes écoulées depuis la frame précédente. En multipliant les déplacements par dt, l'animation devient indépendante du framerate.

graphics.canvas(260, 140, "cycle")
var x = 0

## logique : appelée AVANT draw, dt = secondes depuis la frame précédente
func update(dt)
    x = x + 120 * dt  ## 120 px/seconde, quel que soit le FPS
    if x > 260 then x = 0 end
end

func draw()
    graphics.clear(colors.BLACK)
    graphics.fill(Color(1, 1, 1))
    graphics.circle(x, 70, 20)
end

Tu peux aussi piloter la boucle explicitement avec graphics.run(callback) si tu préfères nommer ta fonction de rendu autrement (update reste appelée avant).

Contexte persistant & modes de fusion

Le contexte de dessin n'est pas effacé automatiquement entre les frames : c'est à draw() de décider. Sans appel à graphics.clear(), le dessin s'accumule d'une frame à l'autre (traînées). Avec une couleur opaque, clear repart d'un fond net ; avec une couleur semi-transparente (alpha < 1), il applique un fondu — le contenu précédent s'estompe peu à peu vers cette couleur.

graphics.blendMode(mode) change la façon dont les dessins se combinent au contenu déjà présent. Le mode additif (blend.ADD), conjugué au contexte persistant et à un léger fondu, produit des lueurs et des traînées lumineuses.

graphics.canvas(240, 160)
global t = 0
func draw()
    graphics.blendMode(blend.ADD)         ## fusion additive (lueur)
    graphics.clear(Color(0, 0, 0, 0.08))  ## fondu léger → traînées
    t = t + 0.05
    graphics.fill(Color(0.2, 0.5, 1))
    graphics.circle(120 + math.cos(t) * 80, 80 + math.sin(t * 1.3) * 50, 14)
end

Couleurs — classe Color

Les couleurs sont des objets Color avec des composantes normalisées dans [0.0, 1.0]. L'alpha est optionnel (défaut : 1.0). Les composantes sont clampées silencieusement.

var rouge = Color(1.0, 0.0, 0.0)       ## rouge opaque
var bleu  = Color(0.0, 0.0, 1.0, 0.5)  ## bleu semi-transparent
print(rouge)                           ## Color(1.000000,0.000000,0.000000,1.000000)

## accès direct aux composantes
print(rouge.r, rouge.g, rouge.b, rouge.a)

3D

Le dessin 3D partage la même boucle et le même état que la 2D. On place une caméra, puis on dessine des formes entre graphics.begin3d(cam) et graphics.end3d(). Comme en 2D, fill/stroke pilotent le remplissage et le contour (fil de fer). Après end3d(), on peut dessiner un HUD 2D par-dessus dans la même frame.

La caméra est un objet (classe Camera, comme Color) : on la crée une fois puis on la pilote par ses méthodes. cam.set_pos(x,y,z) fixe la position, cam.look_at(x,y,z) réoriente vers une cible, cam.move(dx,dy,dz) translate la caméra et sa cible (visée conservée), cam.orbit(angle, rayon [, hauteur]) la place en orbite autour de sa cible (angle en radians).

En 3D, appelle graphics.clear(couleur opaque) au début de chaque frame : un effacement opaque remet à zéro le tampon de profondeur (indispensable au test de profondeur), en plus de la couleur.

Éclairage & texture

Les solides pleins (fill) sont dessinés par instancing avec un shader Blinn-Phong : toutes les formes d'un même type ne coûtent qu'un seul appel de dessin, et chacune garde sa propre couleur. L'éclairage est opt-in : tant qu'aucune lumière n'est déclarée, le rendu est plat (couleur unie). Dès qu'une lumière existe, les faces sont ombrées selon leur orientation.

NomDescription
graphics.ambient(v) ou graphics.ambient(couleur)Lumière ambiante (base uniforme). v ∈ [0,1]
graphics.light("dir", dx,dy,dz [, couleur])Lumière directionnelle ((dx,dy,dz) = direction de propagation) → objet Light
graphics.light("point", x,y,z [, couleur])Lumière ponctuelle à la position (x,y,z) → objet Light
lum.set_dir(x,y,z) · lum.set_pos(x,y,z)Réoriente / repositionne la lumière
lum.set_color(couleur) · lum.enable(bool)Change la couleur / (dés)active la lumière
graphics.texture(img) · graphics.noTexture()Applique une image (handle image) comme texture aux solides suivants ; fill devient une teinte multipliée
Phase 1 : une seule lumière active à la fois (directionnelle ou ponctuelle) + l'ambiante. Le fil de fer, la grille et les lignes ne sont pas éclairés.

Transformer les objets 3D

Les transformations (translate/rotate/scale) agissent aussi sur les solides 3D. translate(x,y,z) place, rotate(deg, ax,ay,az) (ou rotateX/Y/Z) oriente, scale(...) redimensionne. Elles sont cumulatives dans le bloc begin3d/end3d ; push()/pop() servent à isoler la transformation d'un objet (sauvegarde/restauration).

func draw()
    graphics.clear(colors.BLACK)
    graphics.begin3d(cam)
        ## chaque cube dans son propre bloc push/pop
        graphics.push()
            graphics.translate(-3, 0, 0)
            graphics.rotateY(elapsedTime * 60)   ## tourne autour de Y
            graphics.fill(colors.RED)
            graphics.cube(0, 0, 0,  2, 2, 2)
        graphics.pop()
        graphics.push()
            graphics.translate(3, 0, 0)
            graphics.rotate(elapsedTime * 90, 1, 1, 0)  ## axe quelconque
            graphics.fill(colors.SKYBLUE)
            graphics.cube(0, 0, 0,  2, 2, 2)
        graphics.pop()
    graphics.end3d()
end
## la caméra est un objet : créée une fois, pilotée par ses méthodes
global cam = graphics.camera(10, 6, 0,  0, 0, 0)
graphics.ambient(0.2)                         ## éclairage : base ambiante…
graphics.light("dir", -1, -2, -1)          ## …+ un « soleil » directionnel

func draw()
    graphics.clear(colors.BLACK)      ## efface couleur + profondeur
    cam.orbit(elapsedTime * 0.5, 10, 6)   ## orbite autour de la cible (angle en radians)
    graphics.noStroke()
    graphics.begin3d(cam)
        graphics.grid(10, 1)
        graphics.fill(colors.RED)
        graphics.cube(-2, 1, 0,  2, 2, 2)   ## faces ombrées
        graphics.fill(colors.SKYBLUE)
        graphics.sphere(2, 1, 0,  1.2)
    graphics.end3d()
    graphics.draw_text("Ollin 3D", 12, 12, 22)   ## HUD 2D par-dessus
end

    graphics.clear(Color(0, 0, 0))
    graphics.fill(1, 1, 1)
    graphics.circle(px, py, 10)
end

Module image

Le module image permet de charger des textures depuis le disque (ou des données base64) et de créer des render textures pour le rendu hors-écran. Disponible sans import.

graphics.canvas() doit être appelé avant toute opération image — le contexte GL doit exister.

Charger une image

graphics.canvas(240, 140, "image.load")
var tex = image.load("assets/smiley.png")  ## handle {id, width, height} — ressource servie

func draw()
    graphics.clear(Color(0.1, 0.12, 0.2))
    graphics.sprite(tex, 20, 40, 64, 64)   ## 64×64
    graphics.sprite(tex, 120, 25, 96, 96)  ## redimensionné
end
image.load(nom) résout le nom ainsi : en natif, un fichier du disque ; en WASM, d'abord une image chargée via le bouton « Images », sinon une ressource servie par le site (même origine, récupérée automatiquement — ex. image.load("assets/logo.png") relatif à la page). Pour une image embarquée dans le script (portable partout, sans réseau), utiliser image.load_data("png", base64) — voir l'exemple « Image (load_data) » du playground.

Render texture (dessin hors-écran)

graphics.canvas(180, 180, "render texture")
var rt = image.create(120, 120)  ## render texture 120×120

image.begin_draw(rt)             ## on dessine DANS la texture (une fois)
    graphics.clear(Color(0.1, 0.1, 0.15))
    graphics.fill(colors.RED)
    graphics.circle(60, 60, 50)
image.end_draw()

func draw()                      ## puis on l'affiche comme un sprite
    graphics.clear(colors.BLACK)
    graphics.sprite(rt, 30, 30)
end

API

FonctionDescription
image.load(nom)Charge une texture (natif : disque ; WASM : image « Images » ou ressource servie même origine) — retourne un handle {id, width, height}
image.load_data(format, base64)Charge une texture depuis une chaîne base64 (ex. "png", "jpg")
image.create(w, h)Crée une render texture vide w×h — retourne un handle. Requiert un contexte GL actif.
image.begin_draw(handle)Redirige le rendu vers la render texture (équivalent de BeginTextureMode)
image.end_draw()Termine le rendu hors-écran et restaure la cible par défaut
image.draw(handle, x, y [, w, h [, tint]])Dessine la texture ; tint explicite (objet Color) prioritaire, sinon la teinte globale graphics.tint
image.unload(handle)Libère les ressources GPU de la texture
image.begin_pixels(handle)Ouvre l'accès pixel-par-pixel à la texture (à recalculer chaque frame)
image.set_pixel(handle, x, y, color)Écrit un pixel — objet Color ou composantes r, g, b [, a] dans [0,1]. Entre begin_pixels/end_pixels.
image.get_pixel(handle, x, y)Lit un pixel → objet Color
image.end_pixels(handle)Valide les écritures pixel et referme l'accès
graphics.sprite() est l'alternative recommandée à image.draw() : elle respecte la matrice de transformation courante et prend graphics.tint comme teinte (blanc si aucune).

Module window

Le module window expose les dimensions de la zone de rendu courante. Disponible sans import.

PropriétéTypeDescription
window.widthintLargeur de la zone de rendu en pixels
window.heightintHauteur de la zone de rendu en pixels
Le plus souvent tu n'as pas besoin de window : le moteur fournit déjà les globales W et H (largeur/hauteur de la zone de rendu), ainsi que CW et CH (le centre, soit W/2 et H/2), initialisées depuis window avant l'exécution.
## W, H, CW, CH sont fournis par le moteur — pas besoin de window ici
graphics.canvas(W, H, "Titre")  ## fenêtre pleine zone
graphics.circle(CW, CH, 50)  ## cercle au centre
